# Hermes Dispatcher Template

Use Hermes as an intake/router/status layer for the factory.

## Hermes Job File

```markdown
# JOB: <short-name>

Status: pending_vscode_work
Created:
Source type:
Source path or URL:
Target module:

## Goal

## Suggested VSCode Command

## Required Sources

## Approval Needed

## Output Expected
```

## Routing

| Input | Hermes action | Strong-model action |
|---|---|---|
| Brief | Save job, classify | Build module plan |
| Case study | Save raw source | Extract mechanism |
| Video guide | Save raw source | Build implementation SOP |
| GitHub repo | Save URL/snapshot | Extract framework pattern |
| Install request | Ask approval | Prepare install package |
| Deploy request | Ask approval | Runtime-specific execution |

## Hard Stops

Hermes must stop and ask before production edits, deploys, GitHub pushes, payout changes, mass messaging, standalone repo creation, or runtime install.
