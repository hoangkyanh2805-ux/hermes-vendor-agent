# Classification Taxonomy

Classify extracted insights into one or more types.

| Type | Use when |
|---|---|
| `principle` | A durable decision rule or mental model |
| `framework` | A named repeatable structure, e.g. Goal -> Loop -> Tool -> Check |
| `playbook` | A step-by-step operating procedure |
| `checklist` | A verification list for repeated work |
| `template` | A reusable document/table/prompt/schema |
| `sales-asset` | Offer, deck, pitch, pricing, ROI, objection handling |
| `agent-loop` | Goal-driven workflow with tools, checks, stop conditions |
| `folder-structure` | Information architecture for AI/project work |
| `tech-pattern` | Architecture/tooling pattern, integration, deployment model |
| `skill-candidate` | Repeated workflow suitable for a Codex skill |
| `project-map` | Application of source insight to one specific repo/project |

Default priority:

```text
mechanism > framework > project map > reusable asset > implementation task
```
