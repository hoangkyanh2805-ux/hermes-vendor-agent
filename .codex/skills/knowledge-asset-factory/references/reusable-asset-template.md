# Reusable Asset Template

Use this to turn insights into assets future projects can reuse.

```markdown
# <Asset Name>

## Purpose
What repeated problem does this asset solve?

## When To Use
Trigger conditions and example user requests.

## Inputs
What source material or project data is needed?

## Output
What this asset produces.

## Framework
The reusable structure or decision model.

## Template
Copy-ready table/checklist/prompt/schema.

## Adaptation Rules
What changes per project, industry, or client?

## Acceptance Criteria
How to know the asset is useful and complete.
```

Prefer compact assets. Link raw sources separately instead of embedding long source text.
