# Skill Candidate Template

Use this when an idea may deserve its own Codex skill.

```markdown
# Skill Candidate: <name>

## Trigger
What user requests should activate this skill?

## Repeated Job
What repeated work does it do across projects?

## Inputs
Files, URLs, notes, repos, schemas, or user-provided context.

## Workflow
1.
2.
3.

## Outputs
- Plans:
- Docs:
- Assets:
- Code/scripts:

## References Needed
Which templates or domain guides should be bundled?

## Not In Scope
What should the skill not do?

## Recommended Skill Structure
```text
<skill-name>/
  SKILL.md
  references/
```

## Validation
How to test the skill on a real project.
```

Create a skill only when the workflow is likely to repeat across projects. Otherwise keep it as a doc/template.
